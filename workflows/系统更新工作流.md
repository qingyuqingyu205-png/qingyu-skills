### 如果出现问题怎么办？

**问题1：同步后发现有Bug**
```
你："需要回滚到上个版本"
AI：读取备份分支列表 → 选择版本 → 执行回滚 → 5分钟后生效
```

**问题2：不确定某个变更要不要同步**
```
你："先别同步，让我再想想"
AI："好的，变更报告已保存，随时可以继续"
```

**问题3：同步脚本报错**
```
你："sync-to-github.sh报错了"
AI：读取错误信息 → 诊断问题 → 给出解决方案
```

---

### 注意事项

⚠️ **同步前必做：**
1. 确保所有Skills都测试过
2. 确保没有包含个人信息
3. 确保version.json已更新

⚠️ **不要：**
- 跳过备份步骤
- 在未测试的情况下同步
- 手动修改GitHub仓库的文件（会被覆盖）

✅ **建议：**
- 每周五统一检查和同步
- 重大更新前先在测试账号验证
- 保留最近10个备份分支
