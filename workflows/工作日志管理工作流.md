# 工作日志管理工作流

> 查看或更新今日工作日志
> 版本：v1.0

---

## 触发时机

- 用户说"今天的工作日志"
- 用户说"查看工作日志"
- 用户说"看看今天做了什么"
- 在"工作结束"场景中，作为第2个步骤自动执行

---

## 完整执行流程

### 步骤1：检查今日日志是否存在

检查路径：`工作日志/{年}/{月}/{日期}_工作日志.md`

**示例**：
- `工作日志/2026/07/20260706_工作日志.md`
- `工作日志/2026/07/20260707_工作日志.md`

### 步骤2：如果日志已存在

**展示日志内容**：
```
📅 今日工作日志（2026-07-06）

{显示完整日志内容}

---

要更新日志吗？
[1] 是，添加新内容
[2] 否，只查看
```

**如果用户选择更新**：
- 询问要添加什么内容
- 追加到对应section（公众号经营/项目/系统维护等）
- 更新待办清单（检查已完成的打勾）

### 步骤3：如果日志不存在

**自动创建今日日志**

#### 第3.1步：从各位置工作记录提取今日内容

读取以下位置的工作记录文件：
- `Areas/{领域}/04-工作记录/{领域}工作记录_{年月}.md`
- `Projects/{项目}/04-工作记录/{项目}工作记录_{年月}.md`
- `Resources/04-工作记录/Resources工作记录_{年月}.md`

提取今日（{日期}）的所有工作记录。

#### 第3.2步：生成日志内容

**格式**：
```markdown
# {日期} 工作日志

## 公众号经营
- {从公众号经营工作记录提取的今日内容}

## 朋友圈经营
- {从朋友圈经营工作记录提取的今日内容}

## {项目名}
- {从项目工作记录提取的今日内容}

## 系统维护
- {从Resources工作记录提取的今日内容}

---

## 关键决策
- {今天做出的重要决策}

---

## 待办事项
- [ ] {未完成的待办}
- [x] {已完成的待办}

---

**工作时长**：约{X}小时
**主要成果**：{一句话总结}
```

#### 第3.3步：保存日志

保存到：`工作日志/{年}/{月}/{日期}_工作日志.md`

如果文件夹不存在，自动创建。

### 步骤4：更新待办清单

如果是"工作结束"场景调用：

1. **检查今日待办清单**：
   - 读取日志中的"待办事项"部分
   - 对比本次工作记录中完成的内容
   
2. **标记完成项**：
   - 将已完成的待办从 `- [ ]` 改为 `- [x]`
   - 保持未完成的待办不变
   
3. **添加新的待办**（如果有）：
   - 从"下次继续"中提取待办事项
   - 添加到待办清单

### 步骤5：通知用户

**如果是查看已有日志**：
```
✅ 今日工作日志

📅 日期：{日期}
📝 位置：工作日志/{年}/{月}/{日期}_工作日志.md

{显示日志摘要}
```

**如果是创建新日志**：
```
✅ 已创建今日工作日志

📅 日期：{日期}
📝 位置：工作日志/{年}/{月}/{日期}_工作日志.md

汇总了以下位置的工作记录：
- 公众号经营
- Resources

{显示日志摘要}
```

**如果是更新日志**：
```
✅ 已更新今日工作日志

📝 更新内容：
- 已完成待办：2项
- 新增待办：1项
- 新增内容：系统维护板块
```

---

## 特殊情况处理

### 如果今天还没有任何工作记录

```
📅 {日期} 工作日志

今天还没有工作记录。

开始工作后，使用 /qy-record-work 记录工作，
工作日志会自动汇总。
```

### 如果用户想手动添加内容

```
要手动添加内容吗？
- 输入内容直接添加到日志
- 输入 0 取消
```

---

## 与其他工作流的关系

- **与记录工作工作流**：记录工作工作流生成日志，本工作流负责查看和更新
- **在"工作结束"场景中的位置**：第2个执行（在record-work之后，sync-structure之前）
