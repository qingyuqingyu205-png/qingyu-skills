# 结构变更同步工作流

> 当任何文件夹/01-工作流/文件名称或位置发生变化时，必须执行此工作流

---

## 触发时机

1. 新建工作流
2. 新建文件夹
3. 重命名01-工作流/文件夹
4. 移动文件位置
5. 删除01-工作流/文件夹
6. 修改工作流中的路径引用

**关键：用户手动改完后，必须告诉Claude："我改了XXX，帮我同步更新"**

---

## 完整执行流程

### 步骤1：记录变更

AI询问用户：
"检测到结构变更，请告诉我：
1. 你做了什么操作？（新建/重命名/移动/删除）
2. 涉及哪个文件/文件夹？
3. 原来的名称/路径是什么？
4. 现在的名称/路径是什么？"

记录到对应Area的工作记录和更新日志

### 步骤2：分析影响范围

AI执行：
1. 扫描当前Area的所有工作流文件
2. 检查每个工作流中是否引用了变更的路径
3. 检查CLAUDE.md中是否引用了变更的路径
4. 检查README.md中是否引用了变更的路径
5. 检查其他Area的工作流是否引用了这个路径

生成"影响清单"：
- 哪些工作流需要更新？
- CLAUDE.md需要更新哪些部分？
- README.md需要更新哪些部分？

### 步骤3：列出更新计划

AI输出：
"检测到以下文件需要同步更新：

📄 需要更新的工作流：
1. 朋友圈发布工作流.md - 第18行：保存到 02-待发布/ → 需改为：待发布内容/
2. 朋友圈复盘工作流.md - 第12行：读取 03-已发布/ → 需改为：已发布内容/

📄 需要更新的系统文件：
1. CLAUDE.md - 路径速查表需要更新
2. README.md - 文件夹结构图需要更新

是否执行同步更新？"

### 步骤4：执行同步更新（如果用户确认）

对每个需要更新的工作流：
1. 备份当前版本到版本历史
2. 更新路径引用
3. 版本号+0.1
4. 更新日期改为今天
5. 记录到更新日志

对系统文件（CLAUDE.md/README.md）：
1. 备份当前版本
2. 更新路径说明
3. 更新触发词规则（如果工作流改名）

### 步骤5：验证更新

重新扫描所有更新过的文件，检查路径引用是否正确，检查是否有遗漏

### 步骤6：记录完成

写入更新日志：
```
[已处理] 2026-07-06 结构变更同步
- 变更内容：文件夹"02-待发布/"重命名为"待发布内容/"
- 影响范围：朋友圈发布工作流.md (v1.1→v1.2)、CLAUDE.md、README.md
- 执行时间：2026-07-06 15:30
```

通知用户："✅ 结构变更同步完成，已更新2个工作流+2个系统文件"
