# 整理对话记录工作流

> 将AI对话记录整理成"需求-解决方案"格式的素材
> 版本：v1.0

---

## 核心原则

**对话记录的价值在于原文，不在于总结。**

- ✅ 保留AI给出的完整方案原文（一字不改）
- ✅ 提炼用户的需求描述（口语化整理成清晰表达）
- ❌ 不要总结、不要概括、不要系统化改写

---

## 触发时机

**主动触发**：
- 用户提出问题 → AI给出方案 → 用户确认（说"好的"、"可以"、"明白了"，或直接问下一个问题）
- AI主动询问："这个问题已解决，要整理成素材吗？"

**手动触发**：
- 用户说"整理素材"、"整理对话记录"
- 用户输入 `/qy-organize-chat`

**最佳时机**：
- ⚠️ 必须在对话还没被压缩之前整理（否则拿不到原文）
- ✅ 建议每解决1-2个问题就整理一次
- ❌ 不要等一整天结束后再整理

---

## 完整执行流程

### 步骤1：确认整理内容

询问用户：
```
要整理哪部分对话？
[1] 刚才讨论的问题（最近的对话）
[2] 整个对话记录（如果对话还不长）
[3] 我会复制粘贴给你（如果对话已被压缩）
```

### 步骤2：提取需求和解决方案

**提取规则**：

**用户的需求**：
- 从用户的口语化表达中提炼清楚"到底要什么"
- 保留关键的背景信息和上下文
- 整理成清晰的需求描述

**AI的解决方案**：
- **原封不动**复制AI给出的完整回答
- 包括代码、命令、文件内容、步骤说明等
- 不删减、不改写、不总结

### 步骤3：按"需求-解决方案"格式整理

**格式**：
```markdown
# 需求{编号}：{需求标题}

## 需求描述

{用户的问题/需求，提炼自用户的口语化表达}

**背景/上下文**：
{如果有重要背景信息}

**用户原话**：
"{关键的原话片段，保留用户的表达方式}"

## 解决方案

{AI当时给出的完整方案，原封不动，一字不改}

---

**状态**：已解决 / 待验证
**日期**：{YYYY-MM-DD}
```

### 步骤4：确定保存位置和命名

**保存位置**：
- 公众号相关 → `Areas/公众号经营/06-素材库/对话记录/`
- 朋友圈相关 → `Areas/朋友圈经营/06-素材库/对话记录/`
- 项目相关 → `Projects/{项目名}/06-素材库/对话记录/`
- 系统相关 → `Resources/02-系统设计文档/`（特殊情况）

**命名规则**：
- 单个需求：`YYYYMMDD_{主题关键词}.md`
- 多个需求：`YYYYMMDD_{主题}/需求{编号}_{标题}.md`

**需求编号规则**：
- **全局累计编号**，不按天重置
- 如果上次整理到需求9，这次就从需求10开始
- 需求编号记录在素材索引中

**示例**：
```
20260706_Skills系统改进/
├── 需求10_Skills调用确认机制.md
├── 需求11_Skills场景分组.md
├── 需求12_学习用户使用习惯.md
└── 需求13_识别常用工作流.md
```

### 步骤5：AI分析素材价值

**分析1：匹配已有文章**
- 读取素材索引.md中"未使用的文章"
- 判断新需求能否补充哪些已有文章
- 标注："{需求编号} 可用于补充文章X"

**分析2：推导新文章方向**
- 基于新需求内容，推导可能的文章方向
- 生成3-5个具体的文章标题建议
- 每个标题说明：为什么这个需求适合写这个文章

### 步骤6：更新素材索引

在 `素材索引.md` 中：

1. **添加新素材记录**：
```markdown
### [对话记录/20260706_主题/](对话记录/20260706_主题/)

- **来源**：2026-07-06 与AI的对话
- **状态**：待加工
- **格式**：原始问答记录（需求描述 + 完整解决方案）

#### 📋 包含需求（点击查看详情）

1. [需求10：XXX](对话记录/20260706_主题/需求10_XXX.md)
2. [需求11：YYY](对话记录/20260706_主题/需求11_YYY.md)
```

2. **添加文章方向建议**（如果AI分析出新方向）：
```markdown
## 🎯 可产出文章方向（未使用）

### 文章X：《标题》
- **文章状态**：未使用
- **定位**：{文章定位}
- **目标读者**：{目标读者}
- **使用素材**：
  - [需求10：XXX](对话记录/20260706_主题/需求10_XXX.md)
  - [需求11：YYY](对话记录/20260706_主题/需求11_YYY.md)
```

### 步骤7：给用户选择

如果AI分析出新文章方向：

```
💡 可能的新文章方向：

[1] 《手把手教你用AI搭建工作系统》
    - 适合人群：想用AI提升效率但不知道怎么开始的人
    - 使用素材：需求10、11、12

[2] 《如何让AI成为你的长期工作伙伴》
    - 适合人群：用过AI但总觉得不顺手的人
    - 使用素材：需求11、12、13

[3] 不创建新文章，只保存素材

请选择：输入数字（可多选，如：1,2）
```

用户选择后，在素材索引中添加对应的文章规划。

### 步骤8：记录到工作记录

写入对应位置的工作记录：
```markdown
## {日期} {时间} 素材整理

### 本次完成
1. 整理了对话记录：{主题}
   - 提取需求数：{X}个（需求{起始编号}-需求{结束编号}）
   - 保存位置：{路径}
   - AI推荐文章方向：{Y}个

### 下次继续
- 可考虑基于这批素材撰写文章
```

### 步骤9：记录到用户习惯

在 `Resources/02-系统设计文档/用户习惯记录.md` 中记录：

```markdown
### {日期} 内容创作场景

- **用户原话**："{触发整理的话}"
- **识别场景**：内容创作
- **执行的Skill**：/qy-organize-chat
- **整理需求数**：{X}个
- **是否符合预期**：待用户反馈
```

### 步骤10：通知用户

```
✅ 对话记录已整理完成

📝 保存位置：{路径}
📊 整理内容：
- 提取需求：{X}个（需求{起始}-需求{结束}）
- AI推荐文章方向：{Y}个
- 素材索引已更新

{如果有推荐的文章方向}
💡 AI建议可以写{Y}个方向的文章，详见素材索引
```

## 特殊情况处理

### 对话已被压缩，拿不到原文

```
⚠️ 检测到早期对话已被压缩

无法获取完整原文。有两个选择：
[1] 你复制粘贴原文给我（会消耗额外token）
[2] 跳过本次整理

建议：下次在问题刚解决时就整理，避免压缩。
```

### 用户复制粘贴了原文

```
好的，我会基于你提供的原文整理。

请确认：
- 这段对话包含几个需求？
- 主题是什么？

然后我开始整理。
```

### 对话中没有明确的"需求-解决方案"

```
⚠️ 本次对话主要是讨论/探索，没有明确的需求和方案。

是否仍要保存？
[1] 是，作为讨论记录保存（不按需求格式）
[2] 否，不保存
```

---

## 与其他工作流的关系

- **独立执行**：本工作流不依赖其他工作流
- **可配合使用**：整理完素材后，可以调用 `/qy-write-article` 开始写文章

---

## 最佳实践

1. **及时整理**：问题解决后立即整理，不要等
2. **保留原文**：永远不要总结AI的方案，要原封不动
3. **提炼需求**：用户的口语化表达要整理清楚
4. **全局编号**：需求编号全局累计，便于引用
5. **AI推荐**：利用AI分析能力，发现文章方向
