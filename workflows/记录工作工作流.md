# 记录工作工作流

> 在对话结束时，自动总结工作内容并记录到对应位置
> 版本：v1.0

---

## 触发时机

- 用户说"工作结束"
- 用户说"今天工作到这儿"
- 用户说"收尾"
- 用户说"先到这里"

---

## 完整执行流程

### 步骤1：总结本次对话

AI自动回顾本次对话，提取关键信息：

**提取内容**：
- 创建了哪些文件？
- 修改了哪些文件？
- 新建了哪些文件夹？
- 重命名了什么？
- 删除了什么？
- 讨论了什么问题？
- 做出了哪些决策？
- 遇到了什么问题？

### 步骤2：分类到PARA维度

智能判断每项工作属于哪个位置：

**关键词匹配**：
- 提到"公众号" → Areas/公众号经营/
- 提到"朋友圈" → Areas/朋友圈经营/
- 提到具体项目名 → Projects/{项目名}/
- 提到"系统工作流"、"系统设计"、"Skills" → Resources/

**文件路径匹配**：
- 操作了 `Areas/公众号经营/...` → 公众号经营
- 操作了 `Projects/某项目/...` → 某项目
- 操作了 `Resources/...` → Resources

**跨位置工作**：
如果同时涉及多个位置，分别记录到各自的工作记录。

### 步骤3：写入位置工作记录

**记录格式**：
```markdown
## {日期} {时间} 工作记录

### 本次完成
1. {工作项1}
   - {具体内容}
   - {版本号/路径等}

2. {工作项2}
   - {具体内容}

### 遗留问题
- {如果有未解决的问题}

### 下次继续
- {下次要做的事}
```

**保存位置**：
- `Areas/{领域}/04-工作记录/{领域}工作记录_{年月}.md`
- `Projects/{项目}/04-工作记录/{项目}工作记录_{年月}.md`
- `Resources/04-工作记录/Resources工作记录_{年月}.md`

**追加规则**：
- 如果当月文件已存在，追加到文件末尾
- 如果当月文件不存在，创建新文件

### 步骤4：生成今日工作日志

从所有位置的工作记录中提取今日内容，汇总生成。

**汇总逻辑**：
- 读取所有位置的工作记录文件（`*工作记录_{当前年月}.md`）
- 提取今日（{日期}）的所有工作记录
- 按位置分类汇总

**日志格式**：
```markdown
# {日期} 工作日志

## 公众号经营
- {从公众号经营工作记录提取的今日内容}

## 朋友圈经营
- {从朋友圈经营工作记录提取的今日内容}

## {项目名}
- {从项目工作记录提取的今日内容}

## 系统维护
- {从Resources工作记录提取的今日内容}

---

## 关键决策
- {今天做出的重要决策}

---

## 待办事项
- [ ] {未完成的待办}

---

**工作时长**：约{X}小时
**主要成果**：{一句话总结}
```

**保存位置**：
```
D:\一人公司OS\工作日志\{年}\{月}\{日期}_工作日志.md
```

### 步骤5：检测结构变更（如有）

如果本次工作涉及：
- 新建文件夹
- 删除文件夹
- 重命名文件夹/工作流
- 移动文件

自动提示：
```
🔍 检测到结构变更：
+ 新增 {X} 个文件夹
~ 重命名 {Y} 个文件夹
- 删除 {Z} 个文件夹

要执行结构变更同步吗？
[1] 是，立即同步
[2] 先看影响清单
[3] 稍后处理
```

### 步骤6：记录到用户习惯

在 `Resources/02-系统设计文档/用户习惯记录.md` 中记录本次调用：

```markdown
### {日期} 工作结束场景

- **用户原话**："{用户说的话}"
- **识别场景**：工作结束
- **执行的Skill**：/qy-record-work
- **是否符合预期**：待用户反馈
- **命中准确度**：待验证
```

### 步骤7：通知用户

完成后通知用户：
```
✅ 工作记录已完成

📝 位置工作记录：
- {位置1}/04-工作记录/{文件名}.md
- {位置2}/04-工作记录/{文件名}.md

📅 今日工作日志：
- 工作日志/{年}/{月}/{日期}_工作日志.md

{如果有结构变更检测}
🔍 检测到结构变更，是否执行同步？
```

---

## 特殊情况处理

### 如果工作内容为空

如果本次对话只是闲聊或询问，没有实际工作：
```
本次对话没有产生工作记录，无需记录。
```

### 如果对话中途中断

下次对话开始时，询问用户：
```
上次对话未记录工作，要补记录吗？
```

---

## 与其他工作流的关系

- **与工作日志管理工作流**：本工作流生成日志后，工作日志管理工作流负责更新和维护
- **与结构变更同步工作流**：如果检测到结构变更，触发结构变更同步工作流
