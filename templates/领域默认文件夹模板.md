# Areas（领域）默认文件夹模板

> 适用于：需要长期维护的领域
> 使用说明：创建新领域时参考此模板结构

---

## 📁 默认文件夹结构

```
Areas/{领域名}/
├── 01-工作流/            ← 标准操作流程
├── 02-待发布/            ← 准备发布的内容
├── 03-已发布/            ← 已发布内容归档
├── 04-工作记录/          ← 按月记录
├── 05-版本历史/          ← 历史版本
│   ├── 工作流/
│   └── 方案/
├── 06-素材库/            ← 可复用素材
│   ├── 对话记录/
│   ├── 待加工素材/
│   └── 灵感收集/
├── 07-对标研究/          ← 竞品研究
└── 08-运营SOP/           ← 标准化手册
```

---

## 📋 各文件夹用途说明

### 01-工作流/
- 标准操作流程（SOP）
- 例如：`公众号长文工作流.md`、`视频制作工作流.md`
- 配合各类 Skill 使用

### 02-待发布/
- 准备发布的内容草稿
- 等待审核的文章
- 待上线的素材

### 03-已发布/
- 已发布的内容归档
- 按时间或主题分类
- 便于后续检索和复用

### 04-工作记录/
- **{领域名}工作记录_{年月}.md** - 按月记录工作
- 配合 `/qy-record-work` Skill 使用

### 05-版本历史/
- **工作流/** - 工作流文档的历史版本
- **方案/** - 方案文档的历史版本
- 记录重要变更，便于回溯

### 06-素材库/
- **对话记录/** - 整理后的对话素材（配合 `/qy-organize-chat`）
- **待加工素材/** - 原始素材，等待提炼
- **灵感收集/** - 零散的想法和灵感

### 07-对标研究/
- 竞品分析
- 行业研究
- 学习对象的拆解

### 08-运营SOP/
- 标准化操作手册
- 日常维护清单
- 常见问题处理流程

---

## ⚠️ 使用注意事项

1. **领域 vs 项目的区别**
   - 领域：长期持续，没有明确结束时间
   - 项目：有明确目标和期限

2. **文件夹命名规范**
   - 使用数字前缀（01-08）保持顺序
   - 使用中文名称，易读易理解

3. **与 Skill 的配合**
   - `/qy-write-article` - 写文章到待发布/
   - `/qy-organize-chat` - 整理对话到素材库/
   - `/qy-record-work` - 记录工作到工作记录/

4. **领域归档**
   - 停止维护的领域，整个文件夹移动到 `Archives/Areas/{年份}/`
   - 保持原有结构不变

## 📌 典型领域示例

- **课程开发** - 在线课程制作
