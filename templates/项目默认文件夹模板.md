# Projects（项目）默认文件夹模板

> 适用于：有明确目标和期限的项目
> 使用说明：创建新项目时参考此模板结构

---

## 📁 默认文件夹结构

```
Projects/{项目名}/
├── 01-项目概览/           ← 目标、时间线、指标
├── 02-工作流/            ← 项目专属工作流
├── 03-工作记录/          ← 按月记录
├── 04-方案与设计/        ← 需求、设计文档
├── 05-交付物/            ← 最终成果
├── 06-测试与验证/        ← 测试文档
├── 07-版本备份/          ← 重要版本快照
├── 08-相关资料/          ← 参考文档
└── 09-项目复盘/          ← 完成后总结
```

---

## 📋 各文件夹用途说明

### 01-项目概览/
- **项目目标.md** - 项目的核心目标和预期成果
- **项目时间线.md** - 里程碑和关键节点
- **项目指标.md** - 考核指标（配合 `/qy-indicator-driven-project` 使用）

### 02-工作流/
- 项目专属的工作流程文档
- 例如：开发流程、发布流程、测试流程等

### 03-工作记录/
- **{项目名}工作记录_{年月}.md** - 按月记录工作进展
- 配合 `/qy-record-work` Skill 使用

### 04-方案与设计/
- 需求文档
- 设计方案
- 架构图
- 技术选型文档

### 05-交付物/
- 最终交付的成果
- 例如：上线的代码、发布的文章、完成的设计稿

### 06-测试与验证/
- 测试计划
- 测试用例
- Bug 记录
- 验收标准

### 07-版本备份/
- 重要版本的快照备份
- 防止误操作导致数据丢失

### 08-相关资料/
- 参考文档
- 竞品研究
- 学习资料

### 09-项目复盘/
- **项目复盘.md** - 项目完成后的总结
- 经验教训
- 改进建议

---

## ⚠️ 使用注意事项

1. **不是所有项目都需要全部 9 个文件夹**
   - 根据项目规模灵活调整
   - 小项目可以只用 3-5 个核心文件夹

2. **文件夹命名规范**
   - 使用数字前缀（01-09）保持顺序
   - 使用中文名称，易读易理解

3. **与 Skill 的配合**
   - `/qy-indicator-driven-project` - 管理项目指标
   - `/qy-record-work` - 记录工作进展

4. **项目归档**
   - 项目完成后，整个文件夹移动到 `Archives/Projects/{年份}/`
   - 保持原有结构不变
