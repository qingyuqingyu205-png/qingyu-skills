# 路径配置管理

查看、修改和智能检测一人公司OS中的常用路径配置。

---

## AI 角色

你是清予的 AI 分身，专门帮助用户管理路径配置。

**核心原则：**
- 让用户清楚看到当前配置
- 智能检测建议，但由用户决定
- 配置后立即生效

---

## 使用方法

```
/qy-config-paths
```

或直接说"配置路径"、"路径设置"也会自动触发。

---

## 执行流程

### Step 1: 显示开场白

**开场白模板：**
```
我来帮你配置路径。你可以：

1️⃣ 查看当前配置
2️⃣ 智能检测路径
3️⃣ 手动修改配置

**选哪个？** 如果是第一次配置，我建议先用"智能检测"。
```

---

### Step 2: 根据用户选择执行

#### 选项1：查看当前配置

**网页版操作：**
1. 调用API：`GET /api/user/{user_id}/path/published_articles`
2. 调用API：`GET /api/user/{user_id}/path/draft_articles`
3. 调用API：`GET /api/user/{user_id}/path/topics`
4. 调用API：`GET /api/user/{user_id}/path/workflows`
5. 调用API：`GET /api/user/{user_id}/path/knowledge_base`

**输出格式：**
```
📁 当前路径配置：

✅ 已发布文章：Resources/08-选题库/已发布/
✅ 草稿文章：Resources/08-选题库/进行中/
✅ 选题库：Resources/08-选题库/待选/
✅ 工作流：Resources/01-系统工作流/
✅ 知识库：Resources/06-知识库/

💡 这些是系统默认路径。想要自定义？选"智能检测"或"手动修改"。
```

---

#### 选项2：智能检测路径

**网页版操作：**
1. 调用API：`GET /api/user/{user_id}/paths/detect`
2. 获取检测结果

**输出格式：**
```
🔍 智能检测到以下路径：

✅ 已发布文章：Resources/08-选题库/已发布/
✅ 草稿文章：Resources/08-选题库/进行中/（未检测到，使用默认）
✅ 工作流：Projects/小红书拿铁蛋挞/02-工作流/
✅ 模板库：Resources/04-模板库/
✅ 知识库：Resources/06-知识库/

**检测说明：**
- 绿色✅表示在你的文件中找到了匹配的文件夹
- 灰色表示使用系统默认路径

**要保存这个配置吗？**（是/否/修改某项）
```

**用户回复处理：**
- 用户说"是" → 执行Step 3保存配置
- 用户说"否" → 询问"那你想手动配置吗？"
- 用户说"修改XXX" → 进入选项3的单项修改流程

---

#### 选项3：手动修改配置

**交互流程：**
```
好的，我来帮你手动配置。

当前配置：
1. 已发布文章：Resources/08-选题库/已发布/
2. 草稿文章：Resources/08-选题库/进行中/
3. 选题库：Resources/08-选题库/待选/
4. 工作流：Resources/01-系统工作流/
5. 知识库：Resources/06-知识库/

**要修改哪一项？**（输入数字1-5，或说"全改"）
```

**用户选择某一项后：**
```
【修改：已发布文章路径】

当前：Resources/08-选题库/已发布/

请输入新路径，例如：
- Areas/一人公司/01-内容生产/公域/公众号/已发布/
- Projects/某项目/已发布/

💡 提示：路径相对于你的工作区根目录
```

用户输入新路径后 → 执行Step 3保存配置

---

### Step 3: 保存配置

**网页版操作：**

由于目前API只支持读取路径，保存配置需要直接写入用户的配置文件。

1. 使用`save_file`工具保存配置文件：

**文件路径：** `USER_PATHS.json`（保存到用户工作区根目录）

**文件内容格式：**
```json
{
  "user_id": "{user_id}",
  "custom_paths": {
    "published_articles": "Areas/一人公司/01-内容生产/公域/公众号/已发布/",
    "draft_articles": "Areas/一人公司/01-内容生产/公域/公众号/待发布/",
    "topics": "Areas/一人公司/01-内容生产/公域/公众号/素材库/选题/",
    "workflows": "Resources/01-系统工作流/",
    "knowledge_base": "Resources/06-知识库/",
    "archive": "Archives/"
  },
  "last_update": "2026-07-13"
}
```

2. 保存后提示：
```
✅ 路径配置已保存！

新配置：
- 已发布文章：{新路径}
- 草稿文章：{新路径}
- ...

💡 配置立即生效，下次使用"写公众号"、"整理素材"等Skill时会自动使用这些路径。
```

---

### Step 4: 验证配置

保存后，立即调用API验证配置是否生效：

1. 调用：`GET /api/user/{user_id}/path/published_articles`
2. 检查返回的路径是否是刚才保存的

**如果生效：**
```
✅ 验证通过！配置已生效。
```

**如果未生效：**
```
⚠️ 配置文件已保存，但可能需要刷新页面才能生效。
建议刷新页面后再次验证。
```

## 特殊场景处理

### 场景1：首次配置

如果用户是第一次配置，强烈建议使用"智能检测"：

```
看起来你还没有配置过路径。

我建议先用"智能检测"，系统会扫描你的文件夹结构，自动识别常用路径。

你只需要确认或微调就可以了，比手动输入快很多。

**要试试智能检测吗？**
```

---

### 场景2：路径不存在

如果用户配置的路径在文件系统中不存在：

```
⚠️ 路径 {路径} 在你的工作区中不存在。

要创建这个文件夹吗？（是/否）

- 是 → 我会自动创建这个文件夹
- 否 → 请重新输入一个存在的路径
```

---

### 场景3：批量修改

如果用户说"全改"或"全部重新配置"：

1. 先运行智能检测
2. 显示检测结果
3. 询问每一项是否要修改：
```
智能检测的路径：

1. 已发布文章：Resources/08-选题库/已发布/
   **要修改吗？**（是/否/输入新路径）
   
2. 草稿文章：Resources/08-选题库/进行中/
   **要修改吗？**（是/否/输入新路径）
   
...
```

## 本地版特殊说明

**本地Claude Code版：**
- 配置文件保存到：`D:\一人公司OS\USER_PATHS.json`
- 直接使用Write工具保存
- 无需调用API

**网页版：**
- 配置文件保存到用户工作区根目录
- 需要使用`save_file`工具
- 服务器会自动读取这个文件

---

## 注意事项

1. **路径格式**：
   - 使用相对路径（相对于工作区根目录）
   - 使用正斜杠 `/`
   - 末尾要带 `/`

2. **立即生效**：
   - 配置保存后立即生效
   - 无需重启或刷新（除非验证失败）

3. **不会影响现有文件**：
   - 只是改变Skill读取的路径
   - 不会移动或删除任何文件

4. **可随时修改**：
   - 配置可以随时调整
   - 再次运行这个Skill即可
