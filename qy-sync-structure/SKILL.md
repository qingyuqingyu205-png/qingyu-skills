# 结构变更同步

执行结构变更同步工作流，检测文件夹/工作流的变更并同步更新相关引用。

## 使用方法

```
/qy-sync-structure
```

或直接说"检查结构变更"、"同步更新"、"检查变动"也会自动触发。

## 功能说明

当文件夹或工作流发生变更（新建/删除/重命名/移动）时，自动检测影响范围并同步更新所有相关引用。

## 执行流程

### 步骤1：读取所有位置的工作记录

扫描以下位置的工作记录：
```
Areas/公众号经营/工作记录/公众号经营工作记录_{当前年月}.md
Areas/朋友圈经营/工作记录/朋友圈经营工作记录_{当前年月}.md
Projects/{所有项目}/工作记录/{项目}工作记录_{当前年月}.md
Resources/工作记录/Resources工作记录_{当前年月}.md
```

### 步骤2：识别结构变动

从工作记录的"本次完成"中提取关键变动：

**识别关键词**：
- ✅ "新建了XXX工作流" → 结构变动
- ✅ "新建了XXX文件夹" → 结构变动
- ✅ "重命名文件夹：A → B" → 结构变动
- ✅ "删除了XXX文件夹" → 结构变动
- ✅ "移动了XXX到YYY" → 结构变动
- ✅ "XXX改名为YYY" → 结构变动
- ✗ "修改了工作流内容" → 不是结构变动（除非改了路径引用）

### 步骤3：生成变更报告

如果识别出结构变动，生成报告：

```
🔍 检测到结构变化：

【公众号经营】
+ 新增文件夹：对话记录/
~ 重命名文件夹：待加工素材/ → 对话记录/
- 删除文件夹：已加工素材/

【Resources】
+ 新增Skills：qy-organize-chat, qy-daily-log, qy-sync-structure

共计：X处变更
```

### 步骤4：读取结构变更同步工作流

自动读取：
```
D:\一人公司OS\Resources（资源库）\系统工作流\结构变更同步工作流.md
```

按工作流执行以下步骤：

### 步骤5：分析影响范围

扫描所有工作流和系统文件，检查是否引用了变更的路径：

**扫描范围**：
- `Areas/*/工作流/*.md`
- `Projects/*/工作流/*.md`
- `Resources/系统工作流/*.md`
- `CLAUDE.md`
- `README.md`

**检查方法**：
对每个变更的路径，用Grep搜索引用

**生成清单**：
```
📄 需要更新的文件：

工作流：
1. {工作流名称}.md
   - 第X行：{旧路径} → 需改为：{新路径}

系统文件：
1. CLAUDE.md
   - 第X行：{需要更新的内容}

2. README.md
   - 第X行：{需要更新的内容}

共计：X个工作流 + X个系统文件
```

### 步骤6：询问用户

```
检测到 {X} 处结构变化，可能影响 {Y} 个文件。

是否执行自动同步？
[1] 是，立即同步（推荐）
[2] 先看详细清单
[3] 稍后处理（会记录到待办）

请选择：
```

### 步骤7：执行同步（如果用户选择1或2后确认）

**对每个需要更新的工作流**：
1. 备份当前版本到版本历史
2. 更新路径引用
3. 版本号+0.1
4. 更新日期改为今天
5. 记录到更新日志

**对系统文件**：
1. 备份当前版本
2. 更新路径说明
3. 更新触发词规则（如果工作流改名）
4. 更新Skills说明（如果有Skills变更）

### 步骤8：验证更新

重新扫描所有更新过的文件：
- 检查路径引用是否正确
- 检查是否有遗漏
- 检查版本号是否递增

### 步骤9：记录完成

写入更新日志：
```markdown
[已处理] {日期} 结构变更同步
- 变更内容：
  - 新增文件夹：对话记录/
  - 删除文件夹：已加工素材/
  - 新增Skills：qy-organize-chat, qy-daily-log, qy-sync-structure
- 影响范围：
  - CLAUDE.md（更新Skills说明）
  - README.md（更新文件夹结构图）
- 执行时间：{日期} {时间}
```

记录到对应位置的工作记录。

### 步骤10：通知用户

**有变更需要同步**：
```
🔍 检测到 {X} 处结构变化，可能影响 {Y} 个文件

详细清单：
{列出影响清单}

是否执行自动同步？
```

**同步完成**：
```
✅ 结构变更同步完成

📝 更新内容：
- 更新了 {X} 个工作流
- 更新了 {Y} 个系统文件
- 所有路径引用已同步

📋 详细记录：
- {位置}/工作记录/{文件名}.md
```

**无变更**：
```
✅ 未检测到结构变更

系统结构稳定，无需同步。
```

## 特殊情况处理

### 如果没有检测到变更

```
✅ 未检测到结构变更

系统结构稳定，无需同步。
```

### 如果检测到变更但无影响

```
🔍 检测到 {X} 处结构变化

分析结果：无文件引用这些变更，不需要同步更新。
```

### 如果用户选择"稍后处理"

将待处理的同步任务记录到待办：
```markdown
## 待处理的结构同步

### {日期} 检测到的变更
- 新增文件夹：对话记录/
- 可能影响：2个工作流 + 2个系统文件
- 优先级：中
```

## 完成标志

- ✅ 工作记录已读取
- ✅ 结构变动已识别
- ✅ 影响清单已生成
- ✅ 用户已确认处理方式

**如果选择"立即同步"**：
- ✅ 所有受影响文件已更新
- ✅ 版本号已递增
- ✅ 更新日志已记录
- ✅ 更新已验证

**如果选择"稍后处理"**：
- ✅ 待办事项已记录
